# INV-303 — Phishing Email with Malicious Link Detected

| Field | Detail |
|---|---|
| **Alert ID** | INV-303 |
| **Severity** | CRITICAL |
| **Source** | Proofpoint |
| **MITRE** | T1566 - Phishing |
| **Verdict** | True Positive |
| **Score** | 100 / 100 |

## Scenario

A critical priority alert was generated by Proofpoint indicating a phishing email containing a malicious link was received. The email originated from an unauthorized external domain `adminstrust-security.co` targeting user account `asmith` on the internal mail server. The email contained a hyperlink directing targets to a credential harvesting site impersonating legitimate cloud services.

```json
{

  "destination_ip": "192.168.1.45",
  "username": "asmith",
  "hostname": "mail-server",
  "email_sender": "adminstrust-security.co",
  "url": "http://malicious-link.mircosoft-alerts.com"
}
```

## Investigation Steps

1. **Alert Triage:** Reviewed Proofpoint alert regarding an incoming email containing an untrusted external link.
2. **Sender Analysis:** Evaluated the sender domain `adminstrust-security.co` which mimics official security services but has no legitimate corporate baseline.
3. **URL Inspection:** Analyzed the embedded hyperlink `http://malicious-link.mircosoft-alerts.com`. The URL leverages typosquatting ("mircosoft") to deceive users into a credential harvesting scheme.
4. **Impact Assessment:** Identified target host (`mail-server`) and recipient user (`asmith`) to evaluate delivery status and track potential user interaction.
5. **Verdict:** True Positive — Malicious phishing email delivered with intent to harvest user credentials.

## MITRE ATT&CK

- **Tactic:** Initial Access
- **Technique:** T1566 - Phishing
- **Sub-technique:** T1566.002 - Spearphishing Link

## Verdict

TRUE POSITIVE - A credential harvesting phishing campaign successfully bypassed basic filters targeting `asmith` with a lookalike domain and typosquatted URL.

## Containment Actions

- Purge the malicious email from user `asmith`'s mailbox globally across the mail server.
- Block the sender domain `adminstrust-security.co` at the email gateway level.
- Add the malicious domain `malicious-link.mircosoft-alerts.com` to internal DNS blocklists and secure web gateways.
- Reset domain credentials for user `asmith` as a precautionary measure.
- Review proxy and endpoint logs to verify if user `asmith` clicked the hyperlink.
